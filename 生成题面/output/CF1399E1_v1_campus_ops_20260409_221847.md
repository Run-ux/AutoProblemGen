# 校园走廊的节能改造

> Variant: `CampusOps_CountHalves` / `v1` / theme `campus_ops`

## 题目描述

新学期伊始，学校后勤部计划对主教学楼的走廊照明系统进行节能改造。教学楼的结构可以抽象为一棵以校长办公室（编号为 1）为根的树，共有 n 个房间（节点）和 n-1 条走廊（边）。每条走廊连接两个房间，并有一个初始的每日基准耗电量 w。

为了降低整体能耗，后勤部可以对任意走廊执行“调光”操作。每次调光会将该走廊的耗电量更新为当前值除以 2 并向下取整（即 w := ⌊w/2⌋）。同一走廊可以被多次调光。

学校规定，改造完成后，从校长办公室到所有末端教室（即树中的叶子节点）的路径耗电量之和不得超过给定的节能指标 S。路径耗电量定义为该路径上所有走廊当前耗电量的总和。

后勤部希望采用最经济的方案，即使用尽可能少的调光次数。请你计算：在恰好使用最少调光次数的前提下，有多少种不同的调光分配方案？两种方案被视为不同，当且仅当存在至少一条走廊被调光的次数不同（调光操作的先后顺序不影响方案判定）。由于方案数可能很大，请将结果对 10^9+7 取模后输出。

## 输入格式

输入的第一行包含一个正整数 t，表示测试数据的组数。
对于每组测试数据：
第一行包含两个正整数 n 和 S，分别表示房间数量和节能指标上限。
接下来 n-1 行，每行包含三个正整数 u, v, w，表示房间 u 和房间 v 之间有一条走廊，初始耗电量为 w。
数据保证给定的结构是一棵以 1 为根的树，且 1 号节点不是叶子节点（除非 n=2）。

## 输出格式

对于每组测试数据，输出一行一个整数，表示满足条件的调光分配方案数对 10^9+7 取模的结果。

## 数据范围与限制

- 1 ≤ t ≤ 100
- 2 ≤ n ≤ 2000，且所有测试数据的 n 之和不超过 2000
- 1 ≤ S ≤ 10^15
- 1 ≤ w ≤ 10^9
- 时间限制：2.0 秒
- 空间限制：512 MB

## 样例

### 样例 1

**输入**

```text
1
3 4
1 2 4
1 3 4
```

**输出**

```text
1
```

**说明**

初始状态下，根到叶子 2 和 3 的路径耗电量分别为 4 和 4，总和为 8。要求总和不超过 4，至少需要减少 4 点耗电量。每次调光最多减少 2 点（4→2），因此最少需要 2 次操作。满足条件的分配方案只有一种：走廊 (1,2) 和 (1,3) 各调光 1 次，此时两条路径耗电量均为 2，总和为 4。

### 样例 2

**输入**

```text
1
4 6
1 2 3
2 3 3
2 4 3
```

**输出**

```text
3
```

**说明**

初始总路径和为 12。最少需要 2 次操作才能使总和降至 6 或以下。满足条件的分配方案有 3 种：
1. 走廊 (1,2) 调光 2 次，其余 0 次（权重变为 0, 3, 3，总和 6）；
2. 走廊 (1,2) 调光 1 次，(2,3) 调光 1 次，(2,4) 调光 0 次（权重变为 1, 1, 3，总和 6）；
3. 走廊 (1,2) 调光 1 次，(2,3) 调光 0 次，(2,4) 调光 1 次（权重变为 1, 3, 1，总和 6）。

## 说明

1. 本题要求对结果取模，模数为 10^9+7。
2. 方案去重规则：仅统计每条边被分配的操作次数向量 (x_e) 的组合数，不考虑操作执行的先后顺序。若两组分配中每条边的操作次数完全一致，则视为同一种方案。
3. 树的根节点固定为 1 号节点。叶子节点定义为度数为 1 且不为根节点的节点（当 n=2 时，2 号节点为叶子）。
4. 路径耗电量之和的计算仅考虑从根节点到所有叶子节点的路径，中间节点不作为终点计入总和。
